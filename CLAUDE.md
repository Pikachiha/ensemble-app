## デザインルール

### 基本方針
Linear・Notion のようなスッキリしたUIを目指す。
「足す」より「引く」を意識する。

### 絶対にやらないこと
- 絵文字をUIに使用しない（ボタン・ラベル・見出し・メッセージすべて）
- アイコンが必要な場合は lucide-react などのアイコンライブラリを使う
- ボタンやアクセントカラーに紫（purple / violet）を使わない
- 余白を詰めすぎない（窮屈なレイアウトにしない）
- グラデーションを多用しない
- 影（box-shadow）を多用しない

### カラーパレット
- **ベース**：白・オフホワイト（#FAFAFA）・ライトグレー（#F5F5F5）
- **テキスト**：ダークグレー（#111111）・ミディアムグレー（#666666）
- **ボーダー**：ライトグレー（#E5E5E5）
- **アクセント**：ブラック（#000000）または ダークグレー（#333333）1色のみ
- **危険・削除**：レッド系（#EF4444）のみ
- 原則として「黒・白・グレー＋アクセント1色」に絞る

### タイポグラフィ
- フォントは system-ui / -apple-system を使用（特別な指定がない限り）
- 見出しは font-weight: 600 まで（700以上は使わない）
- 本文は font-weight: 400、補足テキストは color でトーンを落とす

### 余白・レイアウト
- カード・セクション間のpadding は最低16px以上
- リスト項目間のgap は最低8px以上
- コンテンツ幅は max-width: 800px 程度を基本とし、中央寄せ

### ボタン
- プライマリボタン：黒背景・白テキスト（シンプル）
- セカンダリボタン：白背景・グレーボーダー・ダークテキスト
- 角�丸は border-radius: 6px〜8px（大きすぎない）
- ホバー時は opacity か background をわずかに変える程度

### コンポーネント
- テーブルは zebra stripe（偶数行の背景色変更）を使わない。ボーダーラインで区切る
- モーダルは中央配置・背景オーバーレイは rgba(0,0,0,0.4)
- フォーム入力欄は border: 1px solid #E5E5E5、focus 時に border-color: #000000

## コーディングルール

### 全般
- コンポーネントは機能単位で分割する（1ファイル1責務）
- ハードコードは避け、定数は constants ファイルにまとめる
- 日本語のコメントでOK

### React
- 関数コンポーネント + hooks を使用（クラスコンポーネント不可）
- props には TypeScript の型定義を付ける
- スタイルは Tailwind CSS を使用。ただしデザインルールに反するクラスは使わない
  - 例：`bg-purple-*` `text-purple-*` `violet` は使用禁止

### Supabase
- 環境変数は `.env.local` に記載（ハードコード禁止）
- RLS（Row Level Security）を必ず有効にする
- クライアントは `lib/supabase.ts` で一元管理

### ファイル構成
```
ensemble/
├── CLAUDE.md         ← このファイル
├── PROJECT.md        ← プロジェクト概要
├── src/
│   ├── components/   ← UIコンポーネント
│   ├── pages/        ← ページ単位のコンポーネント
│   ├── lib/          ← Supabaseクライアントなど
│   ├── hooks/        ← カスタムフック
│   ├── types/        ← TypeScript型定義
│   └── constants/    ← 定数
└── supabase/
    └── migrations/   ← DBマイグレーション
```
